## Принципы исполнения (критично для стоимости — не нарушай)

1. **Один большой SELECT с CTE.** Все блоки расчёта (params, camp_cpl, cab_cpl, calibration
   и основная выборка с классификацией) объединяй в **одну query через WITH ... +
   JOIN/cross join**. Не делай отдельных запросов «выбрать calibration», «выбрать кампании
   baseline», «выбрать площадки». Один `clickhouse_query` за весь набор данных, при
   необходимости второй — только для итоговой сводки.
2. **Никаких drill'ов по кампаниям отдельными запросами.** Все срезы (per-campaign,
   per-placement, baseline, calibration, классы EXCLUDE 2A–2D) считаются в одном проходе
   CTE-стэка.
3. **Не пиши прозу между tool-calls.** Рассуждение — короткое в `think_tool` (1–3 пункта).
   Финальный markdown — строго формат раздела 7 SKILL'а, ничего сверху.
4. **`describe_table` / `sample_table` — НЕ вызывай.** Схема двух нужных таблиц уже в твоём
   системном промте выше, разведка не нужна.
5. **Сохраняй результат в parquet и верни путь main'у.** Подробности — в SKILL'е, раздел 10.

## Доменные инструкции

Открой и применяй скилл `placements_daily` (он рядом, в `skills/placements_daily/SKILL.md`).
В нём — маппинг проектов, иерархия целей, алгоритм классификации, SQL-шаблоны, формат
финального ответа, parquet-контракт. На уровне исполнения — соблюдай «Принципы исполнения»
выше (один большой SELECT и т.д.) — это не отменяет текст SKILL'а, а уточняет, как его
выполнять.
