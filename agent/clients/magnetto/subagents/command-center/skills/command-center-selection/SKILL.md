# Контекст «выделения области» в командном центре

На UI-дашборде `/budget` директолог может обвести мышью карточки кампаний/групп/объявлений или выделить произвольный текст-метрику и задать вопрос. Фронт упаковывает это в специальный markdown-брифинг и шлёт тебе как обычный user-message через `/api/analyze`.

## Как выглядит брифинг

```
Контекст выделения на дашборде:

Где сейчас находится пользователь (родительский контекст):
- кампания «Нити / Конкуренты / РСЯ» (id 115064863, витрина command_center_campaigns, на дату 2026-04-22)

Выделенные карточки (иерархически):
- кампания «115064863» (id 115064863)
    - группа «Резиденция» (id 5503163876, витрина command_center_adgroups, дата 2026-04-22)

Сырой текст, попавший в обведённую область (метрики/абзацы интерфейса):
> Расход 14K ₽  Клики 4410

Вопрос пользователя:
почему мусора 9.5%, что делать?

Подсказка: можешь подтянуть свежие цифры из указанных витрин ClickHouse по перечисленным id и дате.
Анализируй сверху вниз: контекст → кампания → группа → объявление.
Если выделен только текст без карточек — это значит пользователь спросил про конкретный показатель/фразу с экрана.
```

## Три секции — и что с ними делать

### 1. «Где сейчас находится пользователь (parents)»

Это **scope разговора**. Если юзер находится внутри drill-диалога кампании — parents = `[campaign]`. Если внутри диалога объявлений группы — parents = `[campaign, adgroup]`. Даже если он обвёл только часть групп (не саму кампанию), **parents** подсказывают тебе, что вопрос касается именно этой кампании.

Всегда подставляй parents в WHERE основных SELECT'ов, даже если entities не пустой.

### 2. «Выделенные карточки (entities)»

Реальные выбранные объекты (иерархия campaign → adgroup → ad). Показывает, что конкретно обведено. Если entities пустой, а parents есть — юзер обвёл только текст-метрику, а не карточку.

Для каждой entity фронт передаёт inline:
- `kind`: `campaign` / `adgroup` / `ad`
- `campaign_id`, `adgroup_id?`, `ad_id?`
- `label` — короткое имя, которое юзер видит на экране
- `source` — имя витрины (`command_center_campaigns` / `command_center_adgroups` / `command_center_ads`)
- `report_date` — дата snapshot'а, актуального на момент выделения

Это **уже распарсено в markdown** — тебе нужно просто прочитать иерархию и пойти в соответствующую витрину.

### 3. «Сырой текст из области (selected_text)»

Что физически лежало в обведённом прямоугольнике. Бывает:
- Целая метрика-тайл: `"Клики 4410 Расход 14K ₽"` → юзер ткнул в тайл, его интересует конкретная цифра.
- Фраза из tldr-аудита: `"spam выше медианы"` → юзер хочет расшифровку фразы.
- Обрывки карточки: `"Нити / РСЯ Ошибка"` → юзер хочет объяснения слову «Ошибка» (статус).

Используй текст как **подсказку о том, какой аспект юзер хочет знать** (расход? клики? spam? health?), но **сами цифры бери из ClickHouse** — фронт мог показывать устаревшие данные.

## Типичные сценарии

### Сценарий A. Выделена одна кампания + вопрос про неё

```
entities: [campaign id=X]
parents:  []
selected_text: "Расход 120K ₽ Лиды 42"
question: "что с лидами"
```

Действие: один SELECT из `command_center_campaigns WHERE campaign_id=X AND report_date=max(...)`. Сравни `leads_week` vs `leads_prev`, посмотри `health_reason`. Если причина неочевидна — drill в adgroups.

### Сценарий B. В drill-диалоге кампании выделено несколько групп

```
entities: [adgroup id=G1, adgroup id=G2, adgroup id=G3]
parents:  [campaign id=X]
selected_text: ""
question: "сравни эти три группы, кто в красной зоне"
```

Действие: SELECT из `command_center_adgroups WHERE campaign_id=X AND group_id IN (G1,G2,G3) AND report_date=max`. Группы по health, ROAS, clicks. Сравни только эти три.

### Сценарий C. Выделен только текст-метрика без карточек

```
entities: []
parents:  [campaign id=X]
selected_text: "CPC 45 ₽"
question: "это много или нормально?"
```

Действие: вопрос про CPC в контексте кампании X. Возьми из `command_center_campaigns WHERE campaign_id=X` значения `cpc_week` и `cpc_prev`, плюс средний CPC по портфелю (`sum(cost)/sum(clicks)`). Дай сравнение.

### Сценарий D. Нет entities, нет parents — выделен абстрактный текст

```
entities: []
parents:  []
selected_text: "зелёные кампании 17"
question: "кто эти 17"
```

Действие: юзер спрашивает про общий портфельный показатель. SELECT `WHERE health='green'` из campaigns, верни список.

## Правила

- **Всегда сверяй цифры с ClickHouse**, не доверяй `selected_text`. Тайл мог рендериться из фикстуры/кэша фронта.
- **Учитывай `report_date` из брифинга**. Если в брифинге 2026-04-22, а `max(report_date)` в БД = 2026-04-23 — это значит, что пока юзер печатал вопрос, подкатился новый snapshot. Отвечай по текущему snapshot, но **упомяни в ответе**, что данные свежее, чем на экране.
- **Не перечисляй entities обратно юзеру** — он уже знает, что обвёл. Сразу давай анализ.
- **Короткий ответ первой строкой**, потом детали. Типичная длина: 3–8 строк + таблица.
